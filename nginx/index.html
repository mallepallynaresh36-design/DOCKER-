Architecture Overview
How Docker runs Nginx and serves traffic

Nginx :80

Docker runs Nginx inside an isolated container. Port 8080 on your host machine is mapped to port 80 inside the container where Nginx listens. Your static files live in /usr/share/nginx/html inside the container — you mount a local folder to that path using a volume (-v).

  • ✓Nginx — web server serving your HTML/static files
  • ✓Docker — isolates Nginx in a container, no system install needed
  • ✓Port mapping — bridges host port to container port
  • ✓Volume mount — syncs local files into the container live
💡

This guide covers: quick one-liner deploy → custom nginx.conf → docker-compose.yml for production setups.

Prerequisites
Verify Docker is installed and running

You need Docker Engine 20+ installed. Run these commands to verify before deploying.

terminal 
# Check Docker version
$ docker --version

# Check Docker is running
$ docker info

# Pull the official Nginx image
$ docker pull nginx:latest

# List available images
$ docker images | grep nginx

If Docker is not running, start it with: sudo systemctl start docker (Linux) or open Docker Desktop (Mac/Windows)

03
Quick Deploy — One Command
Get Nginx running in under 10 seconds
terminal — basic run 
$ docker run \
  -d \
  --name my-nginx \
  -p 8080:80 \
  nginx:latest

-d — detached mode, runs in background
--name my-nginx — give the container a name you can reference
-p 8080:80 — map host port 8080 → container port 80
nginx:latest — the official Docker Hub image

terminal — with your own HTML files 
$ docker run \
  -d \
  --name my-nginx \
  -p 8080:80 \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  nginx:latest
💡

The :ro flag means read-only inside the container — a security best practice. Your ./html/ folder contains your index.html, CSS, JS files.

✓

Open http://localhost:8080 in your browser — you should see the Nginx welcome page (or your custom HTML).

04
Custom nginx.conf
Override the default config for production setups

Create your own nginx.conf and mount it into the container. This gives you full control over server blocks, gzip, caching headers, proxy settings, and more.

nginx.conf 
server {
    listen       80;
    server_name  localhost;

    # Root directory for static files
    root   /usr/share/nginx/html;
    index  index.html index.htm;

    # Serve SPA (React/Vue) — fallback to index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache static assets 30 days
    location ~* \.(js|css|png|jpg|gif|ico|svg)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip compression
    gzip on;
    gzip_types text/plain application/json application/javascript text/css;

    # Security headers
    add_header X-Frame-Options SAMEORIGIN;
    add_header X-Content-Type-Options nosniff;

    # Custom error pages
    error_page 404 /404.html;
    error_page 500 502 503 504 /50x.html;
}
terminal — run with custom config 
$ docker run -d --name my-nginx \
  -p 8080:80 \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  -v $(pwd)/nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  nginx:latest

# Test config before restarting
$ docker exec my-nginx nginx -t

# Reload config without downtime
$ docker exec my-nginx nginx -s reload
05
Docker Compose
Production-ready multi-container setup

Use docker-compose.yml for production. It's repeatable, version-controlled, and easy to extend with other services like databases or backends.

docker-compose.yml 
version: '3.9'

services:
  nginx:
    image: nginx:latest
    container_name: my-nginx
    restart: unless-stopped
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html:ro
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - ./logs:/var/log/nginx
    environment:
      - NGINX_HOST=localhost
      - NGINX_PORT=80
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost"]
      interval: 30s
      timeout: 10s
      retries: 3
    networks:
      - webnet

networks:
  webnet:
    driver: bridge
terminal — compose commands 
# Start in background
$ docker compose up -d

# Pull latest image and redeploy
$ docker compose pull && docker compose up -d

# Stop everything
$ docker compose down

# Stop and remove volumes
$ docker compose down -v
💡

restart: unless-stopped means Nginx auto-restarts after a server reboot — critical for production deployments.

06
Manage & Logs
Day-to-day operations for your Nginx container
terminal — container lifecycle 
# Check running containers
$ docker ps

# Stop the container
$ docker stop my-nginx

# Start again
$ docker start my-nginx

# Restart (after config change)
$ docker restart my-nginx

# Remove container (must stop first)
$ docker rm -f my-nginx

# Enter container shell (debug)
$ docker exec -it my-nginx /bin/bash
terminal — logs 
# Live logs (follow mode)
$ docker logs -f my-nginx

# Last 100 lines
$ docker logs --tail 100 my-nginx

# Logs since last hour
$ docker logs --since 1h my-nginx

# Access log inside container
$ docker exec my-nginx cat /var/log/nginx/access.log

# Error log
$ docker exec my-nginx cat /var/log/nginx/error.log

# Resource usage
$ docker stats my-nginx
