01 — Docker lifecycle
Container lifecycle flow
Dockerfile→ docker build→ Image→ docker run→ Container (running)→ docker stop→ Container (stopped)→ docker rm→ Gone
Interview tip: Know that stopped containers still exist on disk. Only docker rm removes them. Image stays unless you do docker rmi.

05 — Dockerfile reference
All instructions explained
InstructionPurpose
FROMBase image (first instruction always)
RUNExecute command during build
COPYCopy files from host to image (preferred)
ADDLike COPY + extracts tars + fetches URLs
CMDDefault command (overridable at runtime)
ENTRYPOINTFixed executable (CMD becomes its args)
ENVSet environment variable in image
ARGBuild-time variable (not in final image)
EXPOSEDocument which port the app uses
WORKDIRSet working directory for RUN/COPY/CMD
USERSet user for subsequent instructions
VOLUMEDeclare a mount point
LABELAdd metadata key-value pairs
HEALTHCHECKDefine container health test
ONBUILDTrigger when used as base image
Production-grade Dockerfile example
# Stage 1: Build FROM node:18-alpine AS builder WORKDIR /app COPY package*.json ./ RUN npm ci --only=production # Stage 2: Runtime FROM node:18-alpine WORKDIR /app # Non-root user for security RUN addgroup -S appgroup && \ adduser -S appuser -G appgroup COPY --from=builder /app/node_modules ./node_modules COPY . . USER appuser EXPOSE 3000 HEALTHCHECK --interval=30s --timeout=3s \ CMD wget -qO- http://localhost:3000/health CMD ["node", "server.js"]

06 — Networking
Network commands
docker network lsList all networks
docker network create mynetCreate a bridge network
docker network inspect mynetInspect network details
docker network connect mynet myappConnect running container
docker network disconnect mynet myappDisconnect from network
docker network rm mynetDelete network
docker network pruneRemove unused networks
Network driver types
DriverUse case
bridgeDefault. Isolated network on single host. Containers talk by name on custom bridge.
hostContainer shares host network stack. No isolation. Best performance.
noneFully isolated. No networking at all.
overlayMulti-host (Swarm/Kubernetes). Connects containers across nodes.
macvlanContainer gets its own MAC. Appears as physical device on network.
Key: Containers on the same custom bridge network can reach each other by container name (built-in DNS). Default bridge does NOT have this.

Compose commands
docker compose upStart all services
docker compose up -dStart in background
docker compose up --buildRebuild images then start
docker compose downStop & remove containers
docker compose down -vAlso remove volumes
docker compose psList service containers
docker compose logs -f webFollow service logs
docker compose exec web bashShell into service

Restart policies
PolicyBehavior
noNever restart (default)
alwaysAlways restart, including on Docker daemon restart
on-failureOnly if container exits with non-zero code
unless-stoppedAlways restart unless manually stopped
Production default: Use unless-stopped for services, on-failure for jobs/migrations.
09 — Best practices & optimization
Image size reduction
✓Use alpine/distroless base images
✓Multi-stage builds (exclude build tools)
✓Chain RUN commands with &&
✓Clean package caches in same RUN layer
✓Use .dockerignore aggressively
✗Don't apt-get without --no-cache
✗Don't copy entire repo then delete files
Build cache optimization
✓Order: rarely changes → often changes
✓COPY package.json before COPY . .
✓Use --mount=type=cache in BuildKit
✓Pin base image versions (not :latest)
✗Don't COPY . . early in Dockerfile
✗Don't use ADD when COPY is sufficient
✗Don't install unnecessary packages
Security best practices
✓Run as non-root user (USER directive)
✓Scan images: docker scout or trivy
✓Use read-only filesystem where possible
✓Pass secrets via env vars or secrets
✓Use COPY --chown=user:group
✗Don't hardcode secrets in Dockerfile
✗Don't run containers as root
.dockerignore — always include this
# Version control .git .gitignore # Dependencies (rebuilt inside container) node_modules vendor/ # Build artifacts dist/ build/ *.pyc __pycache__/ # Local env & secrets .env .env.* *.pem *.key # Logs & temp *.log tmp/ .DS_Store # IDE files .vscode/ .idea/

Quick reference — interview essentials
ConceptOne-liner answer
Image vs ContainerImage = blueprint (read-only). Container = running instance.
Layer cachingEach instruction is a cached layer; order from stable→changing.
Multi-stage buildMultiple FROM stages; copy only artifacts to slim final image.
CMD vs ENTRYPOINTENTRYPOINT = fixed binary. CMD = overridable default args.
Volume vs bindVolume = Docker-managed (prod). Bind = host path (dev).
Bridge vs hostBridge = isolated network. Host = shares host network stack.
docker stop vs killstop = SIGTERM (graceful). kill = SIGKILL (immediate).
COPY vs ADDCOPY = simple (preferred). ADD = extracts tars, fetches URLs.
